# hot_upgrade_rollback_timeout

`hot_upgrade_rollback_timeout` specifies the timeout period for rolling back a hot upgrade. If OceanBase Database Proxy (ODP) does not receive a rollback command within the specified time, it performs a rollback.

| Attribute | Description |
|----------|---------|
| Parameter type | INTERVAL |
| Default value | 24h |
| Value range | [[1s, 30d]] |
